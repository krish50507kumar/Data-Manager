# System Architecture

The Data Manager framework follows a clean, decoupled pipeline design, abstracting data access from the processing logic.

**Storage Backend  →  Jobs  →  Results**

1. **Storage Backend**: Handles data ingestion and persistence. Data is loaded from files (CSV, JSON) or memory into a standardized Pandas DataFrame, isolating I/O operations.
2. **Jobs**: The execution layer. The stored DataFrame is passed to specialized job classes (Engineering, Validation, Analytics) which apply operations based on a provided configuration context.
3. **Results**: The output generated by the jobs. This can manifest as in-place transformed data (e.g., cleaned datasets), boolean validation flags with error logs, or comprehensive statistical dictionaries.
